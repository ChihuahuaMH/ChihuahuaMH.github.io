Class hierarchy

An arrangement where classes are organized in a hierarchy of parents and children.

  • Parent / Superclass / Base class

    • Any class with one or more child classes.
    • Use super to reference properties and functions.

  • Child / Subclass

    • Any class that is below another class.
    • Use override to override properties and functions.

  • Root / Top-level class

    The class at the top (or root) of the class hierarchy.

  • Inheritance

    A child class includes or inherits all the properties and methods of its parent class.

  • Abstract class

    It is a class that can’t be instantiated. For example, dogs have many properties common to all dogs, but we can’t create an instance of a non-specific dog, because don’t know its breed, color, size, etc.
