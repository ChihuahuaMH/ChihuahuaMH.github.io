這篇是參考官網 Manhattan 教學 做一點改變。官方那篇比較偷懶， 是用已經寫好的範例檔 manhattan.netgcfg 和 manhattan.jtrrcfg 生成 Manhattan Mobility Model（MMM），但如果要客製化自己的路網，總不能只依靠範例檔案吧！因此這篇是改成下指令的方式練習並去孰悉用到的工具。

Creating the network#

先看原本的 manhattan.netgcfg 作為生成參考：

<grid_network>
    <grid value="true"/>
    <grid.number value="10"/>
</grid_network>

<junctions>
    <no-turnarounds value="true"/>
</junctions>

閱讀 netgenerate 的 docs 看看需要下哪些參數，依個人喜好去做一些調整：

netgenerate -g --grid.number 6 --grid.length 300 --no-turnarounds -o manhattan.net.xml

成功後會有一個 manhattan.net.xml ，用 netedit 打開後長這樣子：

當然也可以用 netedit 畫，還有很多種方法都可以生成路網。

Generating vehicles#

MMM 中的車輛根據指定的 turning ratios 隨機行駛，這個移動類型可以透過 jtrrouter 做到。

jtrrouter 是根據 traffic volumes 和 turning ratios 計算可能使用的路線。 首先要先建立 flow 作為輸入去定義車輛的起點和開始時間。

這裡用到 randomTrips.py對路網生成一組隨機 trips。檔案在 Sumo/tools 中，建議複製一份到目前專案的資料夾下。

python randomTrips.py -n manhattan.net.xml -o flows.xml --begin 0 --end 1 --flows 100 --jtrrouter --trip-attributes "departPos='random' departSpeed='max'"

執行完上面指令後資料夾會多一個 flows.xml，接著要透過這個檔案調用 jtrrouter，一樣在終端機輸入指令：

jtrrouter -n manhattan.net.xml -r flows.xml -o manhattan.rou.xml -T 25,50,25 -A --allow-loops

成功後會多一個 manhattan.rou.xml，接下來是最後一步了！

Setting Configuration#

在專案資料夾下新增 manhattan.sumocfg 設定檔，在 <configuration> 中寫入：

<input>
    <net-file value="manhattan.net.xml"/>
    <route-files value="manhattan.rou.xml"/>
</input>

<report>
    <no-step-log value="true"/>
</report>

立用指令 sumo-gui manhattan.sumocfg 或是直接點開 manhattan.sumocfg 進行模擬！
